# panel-nangong28-lab

用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于收集、整理和存放各类独立的 HTML 页面文件。这些页面可能用于演示、工具、测试或个人项目展示等用途。

每个 HTML 页面均为独立完整的文件，不依赖外部服务或数据库，可直接通过浏览器打开运行。

## 目录结构

```
/
├── pages/          # 存放主要 HTML 页面
├── assets/         # 页面所需的资源文件（CSS、JS、图片等）
├── archive/        # 归档的旧版本或不再维护的页面
└── docs/           # 相关说明文档或附属资料
```

> 目录结构可能随仓库更新而调整，请以实际内容为准。

## 页面归档说明

- 每个 HTML 页面存放于 `pages/` 目录下，命名遵循语义化规则。
- 页面内部引用的资源文件（样式、脚本、图片等）建议存放于 `assets/` 对应子目录中。
- 归档页面会被移入 `archive/` 目录，并保留原有文件名以便追溯。
- 页面内容可能不定期更新或替换，不保证历史版本的兼容性。
